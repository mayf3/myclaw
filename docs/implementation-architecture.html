P0 Phase 0.7 实现评审

更新时间：2026-05-17

总诊断

Phase 0.7 把上一轮“参考 OpenClaw Feishu 但不直接加载”的判断落到代码里：新增 packages/feishu-adapter，承接 config readiness、x-lark signature、verification token、replay guard 和 event normalization。Gateway 也从 321 行热点文件拆成 auth/http/routes/*，主入口降到 90 行。

这仍然不是生产级 Feishu 接入。encrypt payload 解密、WebSocket mode、access policy、outbound rich card、scoped token 和持久 replay window 还没有完成。

核心业务流程图

这张图回答：Feishu webhook 如何通过签名、token、replay、normalize 进入 runtime？

flowchart TD Start[POST /feishu/events] --> Raw[read raw body] Raw --> HasKey{encryptKey configured?} HasKey -->|yes| Signature[x-lark signature before JSON parse] HasKey -->|no| Parse[parse JSON] Signature -->|invalid| Deny[401 invalid signature] Signature -->|valid| Parse Parse --> Encrypt{body.encrypt?} Encrypt -->|yes| Unsupported[501 encrypt not supported] Encrypt -->|no| Token{verification token valid or loopback dev?} Token -->|no| DenyToken[401/403] Token -->|yes| Challenge{challenge?} Challenge -->|yes| Echo[return challenge] Challenge -->|no| Replay{event id seen?} Replay -->|yes| Duplicate[duplicate=true] Replay -->|no| Runtime[receiveMessage rawInbound] Runtime --> AdapterNormalize[feishu-event normalize] AdapterNormalize --> State[(runs/events)]

Review 观察：

  • 优点：有 encryptKey 时签名在 JSON parse 前完成，边界更硬。
  • 优点：encrypted envelope 会先走 encrypt 分支，不会在 token 阶段误判。
  • 优点：event id replay guard 从 route 内部状态抽成 adapter 工具。
  • 风险：没有 encryptKey 时仍可 token-only dev，生产必须配置 readiness。
  • 改进：非 loopback webhook mode 应逐步强制 encryptKey。

关键时序图

这张图回答：一次签名 Feishu challenge 如何从 gateway 到 adapter？

sequenceDiagram participant F as Feishu/Lark participant G as Gateway route participant A as Feishu Adapter participant R as Runtime participant S as State F->>G: POST /feishu/events raw body + x-lark headers G->>A: validateFeishuWebhookSignature(headers, rawBody) A-->>G: ok G->>A: parseFeishuWebhookBody(rawBody) G->>A: validateFeishuVerificationToken(body) alt challenge G-->>F: { challenge } else event G->>A: replayGuard.reserve(eventId) G->>R: receiveMessage(rawInbound) R->>A: normalizeFeishuEvent R->>S: record run/events G-->>F: envelope end

Review 观察：

  • 优点：challenge 不进入 runtime，event 才进入 runtime。
  • 优点：adapter 函数可单测，不依赖 HTTP server。
  • 风险：signature test 只覆盖 hex sha256，未覆盖 encrypted challenge。
  • 改进：补 AES-256-CBC decrypt 和 malformed encrypted payload 测试。
